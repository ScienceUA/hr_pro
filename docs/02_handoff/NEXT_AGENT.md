Что это за проект (кратко)

HR-Pro — учебный инженерный проект по созданию execution-сервиса для автоматизированного поиска и анализа резюме на Work.ua.

Проект развивается по верхнеуровневому плану из 8 пунктов, обозначаемых как L1_1 … L1_8.

Нотация проекта (обязательно к соблюдению)

Верхний уровень плана: L1_1, L1_2, …, L1_8.

Подпункты внутри каждого пункта: 1.1, 1.2, … / 8.1, 8.2, ….

Обозначения вида L6, L1.5 как подпункт пункта 1 запрещены.

Любые ссылки на этапы должны использовать эту нотацию.

Текущий статус проекта

Реализованы пункты плана: L1_1 – L1_5 (включительно).

Проект находится в состоянии готового Local MVP.

Фактический статус зафиксирован в файле STATUS.md.

Ключевые зафиксированные решения (обязательно)

Перед началом любых действий агент обязан принять следующие решения как неизменные:

Local First
Все ключевые компоненты сначала реализуются и стабилизируются локально.

Эталон Local MVP

транспорт: SmartFetcher;

синхронная модель;

библиотека requests;

без прокси;

управление скоростью через явный троттлинг.

Resilient HTTP (L1_3)

модуль реализован как подготовительный;

не используется в runtime Local MVP;

его наличие без подключения является нормой.

Хранение данных

локальное сохранение данных (JSONL) допустимо на этапе Local MVP;

требования об отсутствии долговременного хранения относятся к более поздним этапам.

Запрет преждевременного рефакторинга
Запрещено:

переводить проект на async;

заменять SmartFetcher;

внедрять прокси;

менять архитектуру без согласования.

Все эти решения зафиксированы в файле DECISIONS.md.
