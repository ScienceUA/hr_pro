### Опис стабільного стану

Система надає три HTTP-ендпоінти:
- `POST /preview` — синхронний, повертає `{ total_found, urls }`.
- `POST /analyze` — асинхронний, повертає `202 Accepted` з `session_id`.
- `GET /status/{session_id}` — ендпоінт опитування для отримання стану завдання.

Усі адаптери повністю асинхронні. Блокуючих викликів `asyncio.run()` не залишилося.

> [!NOTE]
> **Відоме обмеження Фази 1:** Статус завдань зберігається в словнику Python у пам'яті (`TASK_STATUS`). Це означає, що статус втрачається при перезапуску контейнера і не може бути розділений між репліками. Це буде замінено на Google Cloud Memorystore (Redis) у Фазі 2 — див. ADR 006.

## [2026-05-01] Завершення Фази 2: Cloud-Native Workflow, Redis HITL та RED-фільтрація

- **Статус:** ✅ СТАБІЛЬНО — Схвалено для проходження етапу Фази 2
- **Git Tag (посилання):** `phase-2-stable`
- **Автор:** Antigravity AI Agent

### Резюме змін

| Компонент | До (Фаза 1.5) | Після (Фаза 2) |
|---|---|---|
| **Стан HITL** | `SearchPayload` у пам'яті | Персистентні сесії на базі Redis |
| **Data Lake** | Лише локальний JSONL | Паттерн Стратегія: GCS / Локальний JSONL |
| **Логіка** | Ручне оцінювання | **Тріо-модель** (🟢/🟡/🔴) |
| **Звітність** | Усі кандидати включені | **RED-фільтр**: RED приховані зі звіту |
| **Масштабування** | Лише локальний процес | Хмарні фонові завдання |
| **Документація** | Англійська/Чернетки | Повна локалізація українською (ADRs/Prompts) |

### Опис стабільного стану

Система тепер підтримує надійний, стейтфул HITL робочий процес:
1. `/preview` зберігає корисне навантаження в Redis і повертає інформацію про сесію.
2. `/analyze` отримує корисне навантаження з Redis і запускає фонове завдання.
3. `/status` надає прогрес у реальному часі та фінальний звіт у форматі Markdown.
4. Усі сирі дані фізично зберігаються в Data Lake (GCS або `out/candidates.jsonl`) перед аналізом.
5. RED-кандидати відфільтровуються з фінального звіту, але залишаються у векторному кеші для ефективності.

### Інструкції з відкату

Щоб повернутися до Фази 1.5 (Локальний стан):
1. **Повернення клієнта Redis:** Повернутися до словника в пам'яті в `app/api/endpoints.py`.
2. **Повернення сховища:** Встановити `USE_GCS=false` та видалити клас `GCSStorage`, якщо необхідно.
3. **Повернення фільтрації:** Видалити перевірку `status != "RED"` у `run_analysis_task`.
4. **Повернення промпту:** Відкотити `SYSTEM_PROMPT` до англійської версії.
5. **Перевірка:** Запустити `tests/test_api.py` і переконатися, що локальний стан все ще працює.
