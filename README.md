# HR-Pro

HR-Pro — учебный проект: агентно-управляемый поиск и анализ резюме на Work.ua.

Порядок усложнения зафиксирован (SOURCE OF TRUTH: `docs/00_context/Development_Plan.md`):
**Local MVP → Agent Logic → Cloud → Proxy**

## 1. Что делает система

1) Пользователь задаёт запрос в свободной форме (вакансия/критерии).
2) Агент разделяет критерии на:
   - обязательные (фильтруемые),
   - опциональные (семантический анализ текста резюме).
3) Агент формирует структурированную команду на поиск.
4) Исполнитель (Local MVP) выполняет поиск и возвращает:
   - `total_found`,
   - список ссылок `resume_urls[]`.
5) Агент загружает и анализирует резюме по ссылкам и формирует shortlist/отчёт.

Результат: ссылки на резюме Work.ua + оценка соответствия + краткий комментарий.

## 2. Роли компонентов

### 2.1 Агент
- принимает сырой запрос пользователя,
- извлекает критерии и формирует команду исполнителю,
- выполняет семантический анализ и ранжирование,
- формирует результат.

Агент не занимается транспортом и не реализует прокси.

### 2.2 Исполнитель (Local MVP)
- детерминированно собирает данные по команде,
- парсит страницы Work.ua,
- сохраняет результат в JSON/JSONL.

Исполнитель не принимает смысловых решений и не оценивает кандидатов.

## 3. Ограничения

- Проект учебный/личный: без перепродажи данных и без публичной публикации резюме.
- Требования «без долговременного хранения» и расширенные ограничения по персональным данным относятся к более поздним пунктам плана и фиксируются отдельно при переходе к ним.

## 4. Текущее состояние и где смотреть детали

- Пункты 1–5 (Local MVP) завершены.
- Эталонный транспорт Local MVP: SmartFetcher (sync, requests, без прокси).
- Resilient HTTP присутствует как подготовительный модуль и не используется в runtime Local MVP.
- Текущий разрешённый фокус разработки: пункт 6 (логика агента и AI-анализ).
