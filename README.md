# HR-Pro

HR-Pro — учебный проект: агентно-управляемый поиск и анализ резюме на Work.ua.

Система строится по принципу **Local First**:
сначала создаётся локальный работающий MVP сбора данных, затем добавляется логика Агента (управление поиском + семантический анализ), затем — контейнеризация/облако, и только после этого — прокси.

---

## 1. Что делает система

1. Пользователь задаёт запрос в свободной форме (вакансия/критерии).
2. Агент преобразует запрос в:
   - обязательные критерии (переводимые в фильтры Work.ua),
   - опциональные критерии (проверяемые по тексту резюме).
3. Агент формирует структурированную команду на поиск.
4. Парсер выполняет поиск и возвращает:
   - `total_found`,
   - список ссылок `resume_urls[]`.
5. Агент загружает и анализирует резюме по ссылкам:
   - семантическое сопоставление,
   - soft skills и контекстные критерии,
   - формирование shortlist/отчёта.

Результат для пользователя: ссылки на резюме Work.ua + оценка соответствия и краткий комментарий.

---

## 2. Роли компонентов

### 2.1. Агент
Агент — центральный управляющий слой:
- принимает сырой запрос пользователя,
- извлекает критерии,
- формирует команду парсеру,
- анализирует полученные резюме и формирует результат.

Агент не занимается транспортом и не реализует прокси.

### 2.2. Парсер (Local MVP)
Парсер — детерминированный исполнитель:
- выполняет поиск по заданной команде,
- парсит страницы Work.ua,
- возвращает структурированные данные.

Парсер не принимает смысловых решений и не оценивает кандидатов.

---

## 3. Ограничения и этика (этапность)

- Проект учебный/личный: без перепродажи данных и без публичной публикации резюме.
- Требования «без долговременного хранения» и дополнительные ограничения по персональным данным относятся к более поздним этапам и будут зафиксированы отдельно при переходе к ним.

## 4. Текущий статус

- Этап 1 (Local MVP) завершён.
- Эталонный транспорт Local MVP: **SmartFetcher (sync, requests, без прокси)**.
- Resilient HTTP присутствует как подготовительный модуль и **не используется в runtime**.
- Текущий фокус разработки: **Этап 2 — логика Агента и AI-анализ**.
